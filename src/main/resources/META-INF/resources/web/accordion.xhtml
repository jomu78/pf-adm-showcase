Accordion This page shows the shared pf-adm styling for PrimeFaces accordions, including default headers, active states, embedded content panels, and semantic box variants.

Use accordions for grouped, collapsible content blocks where each section needs a visible heading and a clear expanded state.

This example keeps the styling inside the shared theme layer so the appearance applies consistently across showcase pages and downstream applications.

Semantic variants are handled in the local `pfadm` layer rather than patching vendored PrimeFaces sources.

Headers and content panels derive from shared `pfadm` tokens for surfaces, borders, focus rings, and semantic colors.

Hover, focus, and active states are centralized so accordions behave consistently with buttons, toolbars, and input groups.

The semantic `box-*` approach is preserved to keep the migration path aligned with AdminFaces-style presentation patterns.
